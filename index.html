Homebrew is the bee’s knees, friends. So far it’s a perfect replacement for MacPorts.

@thillerson twitter.com/thillerson/status/…

Homebrew依赖于XCode，首先需要安装，lion后XCode也是免费的了，直接到App store下载。同时Homebrew也依赖ruby，Mac已经自带ruby。

github中不加入版本控制.gitignore设定

Jan 19th, 2012

这几天研究github page，由于用jekyll本地测试时会生成_site文件夹，所以需要把这个文件夹排除在外再提交到github，到底怎么设置呢？

这就需要用到.gitignore，即不加入版本控制，在git根目录下建立.gitignore，具体设定如下：

tmp.txt         //忽略tmp.txt
*.log           //忽略所有log文件
tmp/*           //忽略tmp文件夹所有文件
log/**/*.log    //忽略log目录下的包括子目录下的所有log文件

其他的一些过滤条件

  • ？：代表任意的一个字符
  • ＊：代表任意数目的字符
  • {!ab}：必须不是此类型
  • {ab,bb,cx}：代表ab,bb,cx中任一类型即可
  • [abc]：代表a,b,c中任一字符即可
  • [ ^abc]：代表必须不是a,b,c中任一字符

由于git不会加入空目录，所以下面做法会导致tmp不会存在

tmp/*             //忽略tmp文件夹所有文件

改下方法，在tmp下也加一个.gitignore,内容为

*
!.gitignore

还有一种情况，就是已经commit了，再加入gitignore是无效的，所以需要删除下缓存

git rm --cached ignore_file

这样就OK了。

从零开始在github上安装octopress

Jan 18th, 2012

Octopress是使用Ruby语言编写的一套建立在jekyll上的博客系统，它有一些很独特的特点：

  1. 使用静态页面，不使用数据库；
  2. 可以轻松使用makedown标记语言编写文章；
  3. 可以与git紧密集成，方便进行博客的版本管理；
  4. 可以于Github Pages集成，不需要单独的web hosting，只要你有github帐号即可。

Octopress 安装

在octopress上面有具体的文档，E文好的可以直接，[点击查看]

我把我的安装过程整理的下，需要的环境：

  • git
  • Ruby 1.9.2

安装Git

首页git是必备的，具体看到github官网上查看帮助文档 ，这个就比较简单了。

安装Ruby

Ruby可以利用RVM或者rbenv来安装，很轻松帮你搞定首页Ruby环境，我是用RVM来安装的。

1.安装 RVM

RVM(Ruby Version Manager)

$ bash -s stable < <(curl -s https://raw.github.com/wayneeseguin/rvm/master/binscripts/rvm-installer)
2.安装Ruby1.9.2
rvm install 1.9.2 && rvm use 1.9.2

安装Octopress

1.安装Octopress
git clone git://github.com/imathis/octopress.git myblog
cd myblog    //这边会有提示信息，yes就行
ruby --version  //Ruby的版本需要在1.9.2版本
2.安装插件等等
gem install bundler //安装相关附属插件等
bundle install
3.安装Octopress 主题
rake install
4. 修改配置

修改配置文件_config.yml，修改url、title、subtitle、author等等，把评论disqus加上，google+、twitter、Facebook等等，统统都加上。 在source下建CNAME文件，

5. 创建github pero

在github上创建一个repository

6. 本地配置github分支
rake setup_github_pages

当命令提示你输入github URL时，输入刚才建立的git地址

7. 写文章
rake new_page["my first blog"]

在”myblog/source/_post”下生成一个**.makedown文件，编辑文章即可。

8. 生成，预览
rake generate
rake preview

在本地 http://127.0.0.1:4000 就可在本地调试页面

9. git提交
rake deploy

直接把_deploy提交到github上，完工！

以后发博客只需要在本地用命令就OK了。

Mac中SSD+HDD优化方案

Dec 8th, 2011

对电子产品毫无抵抗力的我再次出手了，终于下单购买了SSD（OCZ 120G）和8G内存（金士顿4G*2），具体拆机过程就不写了，上篇也发过拆机视频视频: 2011款MBP光驱位升级SSD教程。

至于讨论的比较多的是SSD放在光驱位还是放在硬盘位，有两种说法：

  1. 光驱位的SATA接口的传输速度是1.5G/s,而硬盘位传输速度是3G/s；
  2. 硬盘位的在设计的时候有相应的减震设计，适合传统的机械硬盘，光驱位硬盘托架由于厚度限制没有做相应的减振设计，所以不适合传统硬盘。

可是我的SSD放在光驱位不能用，只能识别，不能抹盘，也不能装系统，所以我还是把SSD放在原来的硬盘位，至于所说的减震设计我也没不太去注意，平时使用的时候小心点。

由于SSD空间有限，对于寸土寸金的SSD空间来说我们能省则省，充分发挥双硬盘的优势才是当务之急，综合网上很多办法，今天就把我的Mac的SSD优化方案分享给大家，给大家提高一个参考。

更换SSD的最大好处就是速度快，最大的缺点就是空间小。如何充分发挥他的优势，弱化他的缺点正是我们优化的最终目的。针对这个就不难理解，把lion系统、bootcamp、应用程序等必须放在SSD上，其余的一些文件可以放在HDD上，必要的做个软连接。

  1. 转移文件 把Lion安装在SSD上，用Timemachine把备份恢复到新系统，在HDD上面对应的建上Documents, Downloads, Music, Movies, Pictures等目录

将想要移动的SSD的文件夹拷贝到HDD对应文件夹下
